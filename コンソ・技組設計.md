## 1. 背景・狙い
- 2024/5開始の5カ年（〜2029/4）スコープのもと、山形県3E（Education/Engineering/Engagement）の自律化を目指す。
- 人10名・予算1億円・大学（寄附講座）という制約の中で、現行の座組（寄附講座運営委員会／宙の種／BSP）をコンソ（入口・意思決定）＋技組（実稼働）という2層にソフトに分離し、現場との連続性を保つ。
- 参画企業が安心して長期コミットできる契約の器と、収益・成果物・知財の扱いを明文化する必要がある。

## 2. BSP 1系・2系のミッション整理
### 2.1 1系（ベニバナサット）の役割
- 目標: 3年以内に山形県初の1U衛星を打ち上げ、得た知見を資産化したうえで打ち上げ後は教育市場（高校・産技短・宙の種biz・実業系のアセンブリ教育）で継続。
- 制約: 県産コンポは時間的に入らない、商用は狙わず、1U制限（例外的に2/3Uは可能）。
- 成果: 打ち上げ実績、統合ログ/試験手順/ノウハウ、教育用BBM/EM/フラットサット。

### 2.2 2系（サクランボサット）の役割
- 目標: 2-3U級の県産標準コンポを構築し、商用ミッションのインテグレーション力と大学連携によるエンジニア供給基盤を作る。
- 1系と並行で標準化・大学調整・市場調査を進め、1系打ち上げ後に本格稼働。
- 成果: 標準モジュール、商用MI実績、大学向け教育（第1階層：1U標準コンポ自作→構造理解、第2階層：2/3U研究立案）。
- 2系の標準は1系教育供給にも活用されることを前提とする。

### 2.3 1系⇄2系の関係
- 1系が生む実績・知見（統合ログ・不具合DB・試験手順・運用SOP）は技組でSSOT/IVVQ/CRに統合 → 2系の標準設計へ反映。
- 2系の標準モジュールは1系教育用BBM/EMへの供給に必要だが、タイミングがずれるリスクがあるのでA: 教育供給を暫定部品で対応、B: 2系の先行開発、C: 教育開始後倒しのいずれかを明文化する。

## 3. コンソ（統合入口＋意思決定）の設計
### 3.1 参画階層（正会員/準会員/オブザーバー）
| 階層 | 特徴 | 技組参加 | NDA/契約 | 公表 | 特典 |
|---|---|---|---|---|---|
| 正会員（Core） | 無償中核稼働・工程/品質/標準に責任 | 可能 | 必須・合意書 | 同意を前提に社名公表可 | 案件優先声がけ、議決権、ロードマップ/標準早期開示、貢献プレミアム |
| 準会員（Delivery） | 成果物納品・SOWベース | 可能 | NDA/SOW | 同意制 | SOW案件優先、HURDLES割引（県助成・条件付き）、案件説明会アクセス |
| オブザーバー（Candidate） | 情報収集/将来候補 | 原則不可（案件化で見直し） | 必要に応じて | 原則非公表 | 定例・説明会参加、将来案件情報 |
- 非技術系（応援層）は宙の種で吸収し、コンソは技術系の入口に特化。
- 準会員の議決権は「対象案件・成果物に限定」。総会/運営委員会で意見表明・意志決定参加を担保。

### 3.2 RACI/権限境界
```
┌────────────────────────────┬────────┬────────┬────────┬────────┐
│ 意思決定項目                │ コンソ │ 技組   │ 大学   │ 宙の種 │
├────────────────────────────┼────────┼────────┼────────┼────────┤
│ 参加企業審査・区分           │ A      │ C      │ I      │ R      │
│ 会員階層の見直し・退会        │ A      │ C      │ I      │ -      │
│ 標準仕様・ロードマップ採否    │ A      │ R      │ I      │ I      │
│ 技術的実施判断（CR/IVVQ）    │ C      │ A      │ I      │ C      │
│ 契約・対外判断・収益入口      │ A      │ C      │ R      │ R      │
│ 知財/成果物帰属              │ A      │ C      │ C      │ -      │
│ 収益・貢献配分ルール         │ A      │ C      │ I      │ -      │
└────────────────────────────┴────────┴────────┴────────┴────────┘
```
- 寄附講座運営委員会は大学側の最高意思決定機関。コンソの総会/運営委員会が知財・収益・移行を決め、対外契約は大学リードだが実質的にはコンソが案件主体。

### 3.3 意思決定機関
- **総会**（年2回想定）: 大学・正会員代表・事務局。議決権は正会員1社1票（大学議長）。準会員は議題限定で議決に参加可。
- **運営委員会**: 大学・正会員・事務局。日常運営/紛争/総会上程。対外契約や公表の個別承認。
- **技術WG**: 標準/CR/IVVQの技術審査。重大変更は運営委員会へ。

### 3.4 会員メリット
- 正会員: 主要WG参加、案件優先声がけ、ロードマップX、貢献プレミアム、視察/交流会、技組との連携優先。
- 準会員: SOW案件優先、HURDLES条件付き割引、技術マッチング、準会員限定発言機会。
- オブザーバー: 定例/説明会/公開イベント、将来案件の一次情報。
- 県外Tier2/3との交流を定期的に設定し、情報・案件の投資機会を創出。

## 4. 契約・収益の集中と移行ストーリー
### 4.1 暫定期の取り扱い
- 暫定期の対外契約主体は大学（寄附講座）。契約・助成・寄附・スポンサー・教育プログラム等の入口はコンソに一本化（いわゆる「コンソ案件」）。技組案件は作らず、技組はコンソからの委託で実務を遂行。
- コントラクトテンプレに「後継法人（コンソ/技組）への移管条項」を設け、現行契約の地位移転・再契約・相手同意・タイミングを明記。
- NDA/秘密情報の扱い（返却・消去・監査）、資産（成果物・データ・設備）の帰属と保守責任、請求・保険・署名権限の切替スケジュールを整理。賠償/保証の責任分界も明文化。

### 4.2 合同会社化・法人移管期
- 技組は任意団体→合同会社→株式会社の3段階で成立。合同会社化時点でも外部入口はコンソのままにし、対外契約は段階的に移管（例: 一部案件の地位移転）。
- 移管にあたっては契約相手の承認、NDA/資産/保険の明示、請求・口座・源泉・署名権限の切り替え、未払/前受の清算を実施。
- 収益プールはコンソが受領し、貢献台帳をもとに技組・先行企業へ委託費・配分。教育/助成/寄附も含む。

### 4.3 株式会社化フェーズ
- 株式会社化後は貢献台帳→持分への変換、先行企業プレミアム（係数1.5〜2.0）、希薄化防止条項、持分・議決権設計を決定。外部資金調達を見据えた法人構成。
- 大学との利益相反・出資関係については関連法人/TLOや個人教員を起点とし、直接の大学株主が問題となる場合は回避策を設計。

## 5. 技組の機能・法人格・報酬設計
### 5.1 役割
- 技組は実働組織として開発・製造・試験（SSOT/IVVQ/CR/統合ログ/試験手順/運用SOP）を担う。技術的実施決定は技組が主体。
- 全案件はコンソが契約したうえで技組に再委託。標準仕様やCR採否、IVVQ審査結果、資産化されたデータを管理。

### 5.2 法人格の道筋
- 任意団体フェーズ（現状）: 貢献台帳運用をスタート。契約・知財・収益ルールの仕組み化。
- 合同会社フェーズ: 収益発生タイミング（教育/商用/助成）で合同会社設立。貢献台帳ベースの配分比率を定め、先行企業を優遇。
- 株式会社フェーズ: 対外スケール/外部資金調達/ガバナンス成熟のトリガーを満たした場合、組織変更。持分・議決権・希薄化防止条項を設定。

### 5.3 先行企業への報酬
- 貢献台帳で工数・成果物・リスク貢献・IPなどを記録。将来収益配分・持分に反映。
- 1系貢献には「創業期プレミアム係数」や優先案件・可視化（標準SSOTの貢献者欄、対外資料掲載）で報いる。
- 非金銭的報酬（HURDLES受講権、交流会参加、優先見積/主担当権）と併用。

### 5.4 株式・資本設計の課題
- 貢献度ベース、均等＋追加出資、段階的付与といった持分設計の検討。
- 希薄化防止策（保護条項、新株発行より譲渡優先、種類株）を明文化。
- 大学・先行企業・スポンサー・外部投資家の仮想的な株主構成を想定しつつ、利益相反やEXITの課題を整理。

## 6. IP・成果物の帰属
| 成果物 | 帰属 | 利用許諾 |
|---|---|---|
| 1系で生まれた統合ログ/試験手順/運用SOP | コンソ共有財産 | コンソメンバーは自由利用可 |
| 2系で生まれた標準仕様 | 技組管理下（コンソ経由でメンバー利用可） | 標準に従えばメンバー利用を可とし、外部には承認付き再許諾 |
| 既存IP | 所有企業 | 個別契約 |
| 共同開発IP | 貢献度に応じて共有 or 技組帰属 | 事前合意 |
- コンソメンバーが商用利用する場合は再許諾や収益還元を明確化。オブザーバーは公開情報のみアクセス可。

## 7. フェーズとゲート（段階的検討）
| フェーズ | 内容 | フェーズでの主要検討事項 |
|---|---|---|
| Phase 0（現状〜6ヶ月） | 参画ルール・貢献台帳・公表基準・契約テンプレ整備 | 対外契約の移管条項・RACI確認、1系/2系のロードマップ固定 |
| Phase 1（合同会社化トリガー前） | 収益モデルの精度確認、IP/成果物ルール、合同会社準備 | 収益原資評価・契約/保険/署名・初期持分草案 |
| Phase 2（合同会社化〜株式会社化） | 技組に案件移管、貢献ベース配分、持分変換 | 合同会社からの技組案件移転スケジュール・外部資金対応 |
| Phase 3（法人化後スケール） | 株式会社化と事業拡大、TRD/MI/REGの体制設計 | 持分・議決権・希薄化防止、分社化/機能移管のルール |

5. 寄附講座運営委員会とコンソ総会/運営委員会の関係・署名権限。

7. 先行企業プレミアムの定量（係数や評価指標）。
8. TRD/MI/REG機能の将来的な分社化タイミングと移行条件。

10. 県助成・HURDLES・交流会の費用/収益モデル。
