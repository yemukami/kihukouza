# 1系（ベニバナサット）

## 目的

山形県初の1U衛星を3年以内に打ち上げ、「山形で衛星が作れる」という実績を創出する。開発・試験・運用を通じて得た知見を資産化し、打ち上げ後は教育市場向けの1Uプラットフォームとして存続する。

## 背景・制約

- 範囲：2024/5開始の5カ年（〜2029/4）
- 3年後打ち上げが最大ミッション（「3年」は便宜的な目標表現。上記5カ年枠内での達成が前提）
- 県産標準コンポは入らない（時間的に間に合わない）
- 商用は狙わない（1Uでは難しい）
- 1U縛り（例外的に2U,3Uも可能）
- 打ち上げ後は2系と並行して教育市場で存続

## 成果物

### 打ち上げフェーズ
- 打ち上げ実績（山形県初）
- 開発・試験・運用の知見（特に打ち上げ試験関連）
- 手順書、教訓、ノウハウの文書化

### 教育展開フェーズ（打ち上げ後）

#### 高校等/宙の種biz
- ペイロードコンペの実施
- 「このペイロードで何ができるか」のアイデア募集
- MIに相談 → 1U調達・実装

#### 産技短/実業系（アセンブリ人材育成）
- **目的**: アセンブリスキルの習得（手を動かすこと自体が目的）
- **内容**: BBMレベル標準コンポーネント製作、構体へのインテグレーション
- **ゴール**: 「作れる」人材
- 手順に従い確実に製作・組立ができる技術者の輩出

※大学連携は2系の機能として実施（下記参照）

## マイルストーン

| フェーズ | 期間 | 目標 |
|---------|------|------|
| 打ち上げ準備 | 〜3年後 | 1U衛星打ち上げ |
| 教育展開 | 打ち上げ後〜 | 教育市場での存続・収益化 |

# 2系（サクランボサット）

## 目的

2,3U級衛星の県産標準コンポーネントを確立し、商用ミッションのインテグレーション能力を獲得する。大学連携による次世代エンジニアの安定供給基盤を構築する。

## 背景・制約

- 範囲：2024/5開始の5カ年（〜2029/4）
- 1系と並行で準備（少し遅れつつ）
- 1系打ち上げ後に本格稼働
- 1系の知見を引き継ぎ、発展させる
- 2系の標準コンポは1系教育用にも提供可能

## 成果物

### 標準化
- 県産標準コンポーネント（OBC、電源系、通信系、姿勢制御系、構体等）
- **サイズ: 基本1U（バッテリー除く）** → 教育用にも2,3U商用にも適用可能
- 再利用可能な設計資産・インターフェース仕様

### 商用MI
- 商用ミッションのインテグレーション実績
- 収益基盤

### 大学連携（研究者育成・エンジニア供給）

**2階層構造**:
```
【第1階層】標準1Uコンポ自作（教育の入口）
├ 目的: コンポーネント構造の理解
├ 対象: 学部生〜修士前半
└ 成果: 「構造を知る」「作れる」
              ↓ 理解したうえで
【第2階層】2,3Uミッション研究（研究本体）
├ 目的: 新規研究テーマの発想・実行
├ 対象: 修士後半〜博士、研究者
└ 成果: 「新しいことを考え、実行できる」
```

**位置づけ**: 産技短/実業系の「アセンブリ人材」とは明確に異なる
- 産技短/実業系: 手を動かすこと自体がゴール（「作れる」人材）
- 大学: 手を動かすことは入口、自律的な研究構想・実行がゴール

**トライアル（3ヶ月後〜）**:
- 教材: 手元のCOTS OBC EM 5基を活用
- 目的: 構造理解（標準コンポ製作ではない）
- 標準コンポ完成後に教材を移行

**本格化（打ち上げ後〜）**:
- 2系標準1Uコンポを教材として使用
- 大学正規カリキュラムとしてのコンポーネント自作教育
  - 教材提供
  - 講師提供
  - コンポーネント構造の理解 → 新規研究テーマの発想・提案・実行へ

## マイルストーン

| フェーズ | 期間 | 目標 |
|---------|------|------|
| 準備 | 現在〜3年後 | 標準化設計、大学調整、市場調査 |
| 本格稼働 | 打ち上げ後〜 | 商用MI、標準コンポ供給 |

## 1系先行企業への報酬設計

### 課題認識

1系企業は「高負荷・高リスク・低収益機会」の状況で3年間稼働する。一方、2系は「準備フェーズ」で商用収益を将来享受する構造になっている。この非対称を放置すると、1系企業の離脱リスクが高まる。

### 基本方針

**1系での貢献は、将来の法人化時に利益配分権/持分として報いる。**

```
【報酬経路】

1系での貢献（工数・成果物・リスク負担）
        │
        ↓ 記録
貢献台帳（定量化・可視化）
        │
        ↓ ルール化
コンソ収入の配分（委託費/優先枠/将来持分）
        │
        ↓ 享受
商用/教育/助成等の原資から還元
```

### 法人化のステップ

```
【Phase 1: 任意団体/準備期】現在〜
- 貢献台帳の運用開始
- 1系貢献の記録・蓄積

【Phase 2: 合同会社化】収益発生時点〜
- 技組を合同会社として設立（対外入口はコンソに一本化し、技組はコンソからの委託で稼働）
- 利益配分比率 = 貢献台帳ベース（出資額とは別に設定可能）
- 1系先行企業は出資額に関係なく配分が厚い設計

【Phase 3: 株式会社化】事業拡大時
- 合同会社から株式会社に組織変更（対外入口は引き続きコンソ）
- 貢献台帳 → 持分比率に変換
- 外部資金調達・事業拡大に対応
```

### 1系先行企業の優遇

| 項目 | 1系先行企業 | 後発参加企業 |
|------|-------------|--------------|
| 貢献評価 | 創業期プレミアム（係数1.5〜2.0） | 通常評価（係数1.0） |
| 利益配分 | 貢献ベース＋優先枠 | 貢献ベースのみ |
| 持分（株式会社化時） | 創業メンバー枠 | 後発参加枠 |
| 案件参加 | 優先権あり | 通常 |

### 貢献台帳の設計（要検討）

| 貢献項目 | 評価方法 | 備考 |
|----------|----------|------|
| 工数 | 人月 × 単価係数 | 無償期間は係数増 |
| 成果物 | 設計書、試験結果、手順書 | 資産化されたもの |
| リスク負担 | 無償稼働期間、責任範囲 | 定性評価を定量化 |
| IP貢献 | 既存IP提供、共同開発IP | 別途IP帰属ルールと連動 |

### 約束事項

1. **貢献は必ず記録する** - 今日からでも貢献台帳の運用を開始
2. **1系貢献は消えない** - 法人化時に必ず反映する
3. **創業期プレミアム** - 早期参加・高リスク期の貢献には係数をかける
4. **収益プール（案）** - 対外収入（受託/助成/寄附/スポンサー/教育プログラム等）はコンソで一元受領し、貢献台帳に基づき配分・再投資する（対象範囲・例外は要検討）

```
【2組織体制】

┌─────────────────────────────────────────────────┐
│ コンソ（窓口機能）                               │
├─────────────────────────────────────────────────┤
│ 1. MI機能（商用）                               │
│    商用ミッション窓口 → 内部で分担指示          │
│                                                  │
│ 2. 教育支援機能                                 │
│    宙の種/高校ミッション伴走 → 内部で分担指示   │
│                                                  │
│ 3. TRD機能（商社的）                            │
│    県外需要入口 → 内部で分担 or 外部商社連携    │
└─────────────────────────────────────────────────┘
                      │
                      ↓ 分担指示
┌─────────────────────────────────────────────────┐
│ 技組（実稼働）                                   │
├─────────────────────────────────────────────────┤
│ ├ OBC担当                                       │
│ ├ 電源系担当                                    │
│ ├ 通信系担当                                    │
│ ├ 姿勢制御担当                                  │
│ ├ 構体担当                                      │
│ └ ミッションペイロード（案件ごと）              │
└─────────────────────────────────────────────────┘
```

### 現在の座組との対応（ソフト接続）

「現状の座組を“そのまま読み替えられる”」ことを優先し、移行後も役割の連続性を保つ。

| 現在の名称 | 役割（要約） | 移行後の位置づけ |
|---|---|---|
| 寄附講座運営委員会 | 最高意思決定（ただしBSP参画企業は入っていない） | コンソの最高決裁枠（暫定期は大学名義で契約/収入を受ける） |
| 宙の種 | 需要喚起＋BSP参画企業の入口（現状は便宜上の入口で、コンソ/技組の実務機能は持たない） | コンソの入口/事務局（需要喚起機能も継続） |
| BSP | 手を動かす場（実装） | 技組（実施の場：コンソ決定に基づき実行） |

### 市場対応

| 市場 | 窓口（コンソ） | 実装（技組） |
|------|---------------|-------------|
| 商用 | MI機能 | 標準コンポ + MI |
| 教育（大学、高校、実業系） | 教育支援機能 | 教材、1U調達、アセンブリ |
| 県外需要 | TRD機能 | 内部調達 or 外部連携 |
| 宙の種biz | 教育支援機能 | ペイロードコンペ→実装 |

## 要議論事項（将来）

### TRD機能と外部商社の関係
- コンソが窓口になりつつ、外部商社に連携する場合のマージンの取り方
- コンソが商社化するのか、あくまで橋渡しか

### MI機能の詳細
- コンソがMIを担う場合のスコープ（要求定義〜インテグレーションのどこまで）
- 技組との分界点

### コンソ/技組の2組織体制と法人格

**現状（暫定）**:
- 大学（寄附講座）主導の暫定運用（契約/資金/利益相反は学内規程に従う前提）
- 暫定期の対外契約主体は大学（ただし案件管理は「コンソ案件」に統一し、技組案件は作らない）
- 助成/寄附/スポンサー/教育プログラム等も、暫定期は大学名義で受けつつ「コンソの対外入口」として一本化する（将来の移管前提）

**想定経路**:
```
コンソ: （暫定：大学が契約主体）→ 任意団体 → 一般社団法人（対外入口・窓口）
技組:   任意団体 → 合同会社 → 株式会社（実稼働の受け皿 ※対外入口にしない）
```

**契約面で整理が必要な項目（大学→コンソ移行）**:
- 既存契約の移管方式（地位移転/更改/再契約）と相手先同意・タイミング
- 暫定期契約への「後継法人への移管」条項の入れ方（テンプレ化）
- NDA/秘密情報の取扱い（大学→コンソでの継承、返却・消去・監査権限）
- 成果物・データ・設備の帰属（大学保有/貸与/譲渡、保守費用・管理責任）
- 請求/支払（口座・税務・源泉等）の切替と、未払/前受の精算
- 賠償/保証/保険の責任分界（誰が契約責任を負うか、上限、保険加入主体）
- 署名権限（誰がいつ何を決裁し署名するか）と、学内稟議の最短導線

**検討事項**:
- コンソ法人化（任意団体→一般社団法人）のトリガー（対外契約件数/金額、資金受領、会計・監査、責任体制）
- 合同会社化/株式会社化のトリガー（対外入口はコンソのまま、技組は実稼働の受け皿）
- コンソ↔技組の契約関係（業務委託/再委託範囲、原価・マージン、責任分界）
- 人員の兼務可否
- 貢献台帳 → 配分/優先権/（必要なら）持分への変換ルール
